Project Report

System documentation and design overview

Overview

StockBase is a database-driven inventory and order management system built as a first-year group project at Technological University Dublin. It allows users to manage items and orders through a multi-page web interface connected to a live Supabase (PostgreSQL) database.

The system was designed to demonstrate full CRUD functionality across multiple database tables, with a focus on clean UI, real-time data updates, and a consistent user experience across all pages. Each page handles its own database operations independently using the Supabase JavaScript client.

The project was built collaboratively across five team members, with each member taking ownership of a specific section of the system.

Features

  • Manage items (Add, Edit, Delete)
  • Manage orders (Add, Edit, Delete)
  • Dashboard statistics pulled live from the database
  • Real-time data updates via Supabase JavaScript client
  • Consistent navigation and UI across all pages

The system uses a consistent navigation bar across all pages, allowing users to move between the Home, Items, Orders, and Report pages easily. This improves usability and ensures a smooth user experience.

CRUD Functionality

OperationDescription
CreateAdd new items and orders via forms on each page
ReadDisplay live data from Supabase in formatted tables
UpdateEdit existing records using pre-filled forms
DeleteRemove records from the database with confirmation

The system fully supports CRUD operations. Users can add, edit, and delete both items and orders through forms. When editing, the form is pre-filled with existing data to make updates easier.

Validation is also implemented to ensure that all required fields are filled before submission. This prevents invalid or incomplete data from being added to the database.

Database Tables

The system uses three related tables: categories, items, and orders. The categories table stores category names, while the items table links to categories using a category_id field. This creates a one-to-many relationship, where one category can contain multiple items.

The orders table links to the items table using item_id. This allows the system to track which items are being ordered. One item can appear in multiple orders, forming another one-to-many relationship.

These relationships ensure that data is organised efficiently and avoids duplication. The structure allows the system to display meaningful information such as item names and category names instead of just IDs.

TableDescription
ItemsStores item ID, name, category, price and stock quantity
OrdersStores order ID, customer ID, item ID, quantity, date and status
CategoriesStores category IDs and names linked to items

Challenges

  • Connecting the frontend to Supabase — configuring the correct URL, API key and understanding how the JavaScript client works took initial research and testing.
  • Handling async data — ensuring the page waited for Supabase responses before rendering the table required careful use of async/await throughout the code.
  • Managing edit vs add logic — using a single form for both adding and editing meant tracking state carefully so the correct database operation was triggered.
  • Ensuring consistent styling across all pages — small differences in CSS caused layout issues, which were fixed by standardising navigation styling.

Conclusion

In conclusion, StockBase demonstrates a working database-driven web application with multiple related tables, full CRUD functionality, and a consistent user interface across all pages. The project shows how Supabase can be used to store, manage, and display live data in a simple and effective way.
